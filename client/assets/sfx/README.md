# STARFRAG sound effects

The client plays sounds by name via `playSfx(name)` in `client/js/game.js`. Drop
mp3s here and they play automatically — no code change needed.

| file             | plays when                                      |
|------------------|-------------------------------------------------|
| `shoot.mp3`      | you fire the pulse carbine                       |
| `whoosh.mp3`     | plasma-bolt launch layer, under the shot (quiet) |
| `reload.mp3`     | you start a reload                               |
| `hurt.mp3`       | you take damage                                  |
| `death.mp3`      | you die                                          |
| `spawn.mp3`      | you (re)spawn                                    |
| `impact.mp3`     | your plasma bolt splashes on a wall              |
| `hitmarker.mp3`  | your shot lands on an enemy (server-confirmed)   |
| `killconfirm.mp3`| your shot gets the frag                          |

These are generated by BMO's `gen-sfx.mjs` (ElevenLabs, in the sprite-forge
repo) and are gitignored (binary blobs) — regenerate rather than commit. To add
a new sound: drop `foo.mp3` here and call `playSfx('foo')` at the moment.

The combat-feel sounds (added in the feel pass) were generated with:

```
G=/home/claudebot/repos/sprite-forge/generators/gen-sfx.mjs
node "$G" "short sharp plasma bolt impact on metal hull, sizzling electric spark zap, punchy sci-fi, single hit, dry" impact.mp3 0.6 0.55
node "$G" "crisp short high-pitched hit confirmation tick, clean digital blip thunk, video game hitmarker, single, dry" hitmarker.mp3 0.5 0.5
node "$G" "fast plasma energy bolt launch whoosh, sci-fi pulse laser fwip, short airy zap, single" whoosh.mp3 0.5 0.5
node "$G" "heavy satisfying kill confirmation, deep bass thunk with a metallic crunch, sci-fi frag, single, dry" killconfirm.mp3 0.7 0.55
```

Audio only plays after the human clicks "ENTER THE ARENA" (browsers block
autoplay). Bots run muted — that's expected.
